## 二、关键规则机制

### 2.1 底盘功率超限 (5.1.4节)
- 缓冲能量上限 Q = **60J**
- 检测频率 10Hz (100ms周期)
- Z = Z - (Pr - Pl) × 0.1
- Z ≤ 0 → 底盘断电5秒
- Z > Q → Z = Q (上限截断)
- 战亡时缓冲能量重置为Q

### 2.2 射击热量超限 (5.1.3节)
- 每发17mm弹丸: Q1 += 10
- Q1 > Q0 → 发射机构锁定 + 图传断开
- Q1 ≥ Q2(Q0+100) → 当局永久锁定
- 冷却按10Hz结算: 每秒冷却值/10
- Q1 ≤ 0 时解锁

### 2.3 射击初速度超限 (5.1.2节)
- 17mm弹丸 V0 = 25m/s
- 0 < V1-V0 < 5 → 锁定15秒
- 5 ≤ V1-V0 < 10 → 锁定20秒
- V1-V0 ≥ 10 → 当局永久锁定

### 2.4 能量机制 (5.6.7节) — 2026新增
- 初始底盘能量: **20000**, 上限: **40000**
- 能量 ≤ 0 → "节能"状态, 底盘功率基础值 = **35W**
- 能量 ≥ 25000 → 功率 ×**125%**, 最高不超过200W
- 超级电容充电: 每1J差值 → 底盘能量+8
- 底盘能量消耗: Chassis接口每输出1J → 底盘能量-1

### 2.5 装甲/超容离线 (5.1.5.2节)
- 每500ms检测周期扣除: Value = 4 × N (N为离线模块数)

### 2.6 伤害值 (5.1.1节)
- 17mm弹丸对机器人: **20**点伤害
- 17mm弹丸有效检测速度: >12m/s

## 三、代码架构概览

### 任务调度 (FreeRTOS, 6个任务)
| 任务 | 周期 | 功能 |
|------|------|------|
| DecodeTask | 1ms | 遥控器/电容/裁判系统/IMU/视觉解码 |
| MotorUpdateTask | 2ms | CAN反馈+PID+功率限幅 |
| CanTransimtTask | 2ms | CAN控制帧发送 |
| ControlTask | 2ms | 运动学+云台+防卡弹 |
| UiSendTask | 30ms | 裁判系统UI |
| ArmTask | 20-300ms | DM电机安全启动 |

### 硬件配置
- **CAN1**: M3508×4(底盘) + M2006×2(副履带) + M2006×1(拨盘) + M6020×1(云台Yaw)
- **CAN2**: M3508×2(摩擦轮) + M6020×4(云台) + DM电机×3(轮腿)
- **UART4**: 视觉上位机 (460800波特率)
- **UART5**: 超级电容 (115200)
- **UART1**: 裁判系统 (115200)
- **UART6**: 遥控器DBUS (921600)

### 功率管理系统
- **PowerLimiter**: RLS在线辨识+动态功率分配
- **SUPERCAP**: 超级电容通信与能量管理
- **flag_shoot模式**: SUPER(+150W) / NORMAL(-10W)

## 四、规则对照代码问题清单

### 🔥 高优先级
1. **射击热量未主动控制**: 代码读取热量值但未用于调节射速，完全依赖裁判系统锁定
2. **弹速未标定**: 7000RPM摩擦轮对应的实际弹速未验证是否超25m/s上限

### 🔶 中优先级
3. **射击初速度超限无应对**: 代码只是被动接收弹速数据，不调节摩擦轮转速

### 🔷 低优先级
4. **RLS参数缺乏上限保护**: k1/k2在线估计无上限截断
5. **k3估计在比赛中几乎不触发**: 需要空载条件才能更新
6. **裁判系统UI默认ID**为蓝方标准3号机器人(`UI_Data_RobotID_BStandard3`)，实际比赛需确认

> 注：底盘功率上限已由裁判系统 `chassis_power_limit` 综合计算（含等级、底盘类型、底盘能量等因素），代码直接使用该值，无需自行追踪。

## 五、Power_limit.cpp 自适应功能分析

### 三层自适应
1. **RLS在线辨识 k1, k2** (电机损耗系数) — ✅ 带遗忘因子λ=0.999
2. **指数平均估计 k3** (静态损耗) — ✅ 但仅空载时触发
3. **能量反馈动态功率上限** — ✅ 基于电容能量/裁判系统缓冲能量的平方根反馈，裁判系统 `chassis_power_limit` 已综合底盘能量等因素

### 潜在问题
- RLS λ=0.999遗忘极慢 (~2秒时间常数)
- 参数无上限保护
- k3在比赛中几乎无法更新
